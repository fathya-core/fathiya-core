# FATHIYA CORE — Cluster Promotion Plan v0

## الحكم
بعد Complete Coverage وPolicy Layer، لا نبدأ Playbooks مباشرة من كل بطاقة.
نبدأ بترقية العناقيد الأكثر تكرارًا وقيمة.

## الهدف
تحويل المعرفة المتفرقة إلى Playbook Candidates بدون فقدان السياق.

## Promotion Ladder
Raw Item → Knowledge Card → Cluster → Candidate → Playbook → Workflow/Adapter

## شروط الترقية من Cluster إلى Playbook Candidate
1. تكرار النمط في أكثر من مصدر.
2. وجود قيمة تشغيلية واضحة.
3. وجود مخرج Artifact قابل للتكرار.
4. وضوح السياق: Knowledge / Lab / Target / Crypto / Automation.
5. قابلية الفصل بين الفهم والتنفيذ.
6. وجود Stop Conditions.
