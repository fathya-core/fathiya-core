## What Remains Blocked

The following actions require separate explicit approval and must NOT proceed unattended:

| Blocked action | Reason | Gate |
|---|---|---|
| Send email or external messages | External side effect | `approval_external_message` |
| Call webhooks | External side effect | `approval_webhook_or_workflow` |
| Activate n8n or Zapier workflows | External side effect with live triggers | `approval_webhook_or_workflow` |
| Execute trades or portfolio mutations | Financial execution | `approval_market_execution` |
| Live security scanning or probing | Target interaction | `approval_target_specific_external` |
| Credential access or rotation | Sensitive operation | Requires explicit operator command |
| Destructive actions (delete repos, drop tables) | Irreversible | Requires explicit operator command |
| Third-party target activity | External scope | `approval_target_specific_external` |
| Production deployment | External side effect | Requires deployment policy |

Items routed to the Approval Queue remain in `waiting_approval` status until the operator explicitly approves them.
